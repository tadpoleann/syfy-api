syfy API Documentation

Overview

The syfy API allows movie enthusiasts to read information about different movies. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Design Criteria

User Stories

  • As a user, I want to be able to receive information on movies, directors, and genres so that I can learn more about movies I’ve watched or am interested in.
  • As a user, I want to be able to create a profile so I can save data about my favorite movies.

Essential Features

  • Return a list of ALL movies to the user
  • Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user
  • Return data about a genre (description) by name/title (e.g., “Horror”)
  • Return data about a director (bio, birth year, death year) by name
  • Allow new users to register
  • Allow users to update their info (username, password, email, DoB)
  • Allow users to add a movie to their list of favorites
  • Allow users to remove a movie from their list of favorites
  • Allow existing users to deregister
  • Allow users to see which actors star in which movies
  • Allow users to view more information about different movies, such as the release date and the movie rating

Endpoints for REST API

Description URL HTTP method Query Parameters Request body data format Response body data format
Return a list of all movies to user /movies GET None None A JSON object that holds the data about all of the movies.
Return data about a single movie by title to user /movies/[Title] GET /movies/:title None A JSON object that holds data about about the single requested movie.
Example:
{
"title": "Sharknado",
"description": "Sharknado is a ...",
"director": "Anthony C. Ferrante",
"genres": "Sharks",
"image": "https://m.media-amazon.com/images.."
}
Return data about all genres /genres GET None None A JSON object that holds data about all the genres.
Return data about a genre (description) by name (e.g., “Horror”) /genres/[Genre] GET /genres/:genre None A JSON object that has data about a specific genre.
Example:
{
"genre": "Sharks",
"description": "A movie with a lot of sharks."
}
Return data about all directors. /directors GET None None A JSON object that has data about all the directors.
Return data about a director (bio, birth year, death year) by name. /directors/[Name] GET /directors/:name None A JSON object that has data about a movie's director.
Example:
{
"name": "Anthony C. Ferrante",
"biography": "an American film director, producer, and writer...",
"birthday": "January 01, 1962"
}
let users view information about different actors /filmstars GET None None A JSON object that has data about film stars.
Allow users to see which actors and actresses star in which movies /filmstars/[Name] GET /filmstars/:name None A JSON object that returns data about a specific film star.
Allow users to add a movie to their list of favorites /users/[Username]/favorites POST /users/:username/favorites A JSON object that has data about the movie to add to favorites. A text message indicating that the movie has been added to the user's favorites.
Allow users to remove a movie from their list of favorites /users/[Username]/favorites/[Title] DELETE /users/:username/favorites/:title None A text message indicating that a movie has been removed from the user's favorites.
Allow new users to register /users POST None A JSON object holding data about a user to register.
Example:
{
"name": "user1",
"birthday": "January 1, 1905"
}
 		A text message response indicating that a user has been registered.
Allow users to update their user info /users/[Username] PUT /users/:username None A text message indicating that the user has been successfully updated.
Allow existing users to deregister /users/[Username] DELETE /users/:username None A text message indicating that an existing user has been deregistered.
